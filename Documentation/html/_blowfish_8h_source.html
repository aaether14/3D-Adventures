6 // BLOWFISH ENCRYPTION ALGORITHM
7 //
8 // Encryption and Decryption of Byte Strings using the Blowfish Encryption Algorithm.
9 // Blowfish is a block cipher that encrypts data in 8-byte blocks. The algorithm consists
10 // of two parts: a key-expansion part and a data-ancryption part. Key expansion converts a
11 // variable key of at least 1 and at most 56 bytes into several subkey arrays totaling
12 // 4168 bytes. Blowfish has 16 rounds. Each round consists of a key-dependent permutation,
13 // and a key and data-dependent substitution. All operations are XORs and additions on 32-bit words.
14 // The only additional operations are four indexed array data lookups per round.
15 // Blowfish uses a large number of subkeys. These keys must be precomputed before any data
16 // encryption or decryption. The P-array consists of 18 32-bit subkeys: P0, P1,...,P17.
17 // There are also four 32-bit S-boxes with 256 entries each: S0,0, S0,1,...,S0,255;
18 // S1,0, S1,1,...,S1,255; S2,0, S2,1,...,S2,255; S3,0, S3,1,...,S3,255;
19 //
20 // The Electronic Code Book (ECB), Cipher Block Chaining (CBC) and Cipher Feedback modes
21 // are used:
22 //
23 // In ECB mode if the same block is encrypted twice with the same key, the resulting
24 // ciphertext blocks are the same.
25 //
26 // In CBC Mode a ciphertext block is obtained by first xoring the
27 // plaintext block with the previous ciphertext block, and encrypting the resulting value.
28 //
29 // In CFB mode a ciphertext block is obtained by encrypting the previous ciphertext block
30 // and xoring the resulting value with the plaintext
31 //
32 // The previous ciphertext block is usually stored in an Initialization Vector (IV).
33 // An Initialization Vector of zero is commonly used for the first block, though other
34 // arrangements are also in use.
